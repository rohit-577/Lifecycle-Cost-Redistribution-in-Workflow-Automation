# Contributing to Lifecycle Cost Redistribution in Workflow Automation

We welcome contributions that improve the reproducibility, clarity, and scope of this benchmark.

## How to Contribute

1. **Fork the repository** and create a feature branch.
2. **Make your changes** with clear commit messages.
3. **Run the verification checks** if applicable.
4. **Submit a pull request** describing the changes and their rationale.

## Types of Contributions

- **Replication results**: Submit logs and measurements from independent replication runs.
- **Task extensions**: Propose new benchmark tasks covering additional coordination patterns.
- **Platform additions**: Contribute implementations on additional visual or AI platforms.
- **Documentation improvements**: Clarify methodology, fix errors, add examples.
- **Bug fixes**: Correct errors in analysis scripts, data, or documentation.

## Guidelines

- All contributions must be reproducible and documented.
- Data contributions must include raw measurements, not just aggregates.
- Code contributions must follow the existing style (PEP 8 for Python).
- Do not fabricate or modify experimental results.
- Cite the original paper when discussing findings.

## Code of Conduct

All contributors must adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).
